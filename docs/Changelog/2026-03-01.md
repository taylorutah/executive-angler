# Changelog — 2026-03-01

## Phase 1: Initial Build

Built the entire Executive Angler MVP from scratch.

### Scaffolding
- Initialized Next.js 16.1.6 project with TypeScript 5, React 19, App Router
- Installed and configured: Tailwind CSS v4 (CSS-based `@theme inline`), Supabase (`@supabase/ssr` + `@supabase/supabase-js`), Mapbox GL JS, Framer Motion, Lucide React, sharp, Vercel Analytics + Speed Insights
- Custom theme: brand colors (forest, river, cream, gold), fonts (Playfair Display headings, Source Sans 3 body), custom CSS utilities
- Deployed to Vercel with auto-deploy from `main` branch

### Database (supabase/schema.sql)
Created 10 tables (Phase 1) with full RLS policies, indexes, and update triggers:
- Content: `destinations`, `rivers`, `lodges`, `guides`, `fly_shops`, `articles`, `species`
- User: `user_favorites`, `user_profiles`, `reviews`
- All content tables: public read, no public write
- User tables: scoped to `auth.uid() = user_id`

### Pages Built (Phase 1)
- **Homepage** (`/`) — full-bleed hero with Madison River photo, featured destinations grid (6), featured rivers (4), featured lodges (3), latest articles (3), CTA section

- **Madison River** (`/rivers/madison-river`) — hero, overview, interactive Mapbox map with access points as markers, monthly hatch chart, regulations, nearby lodges

- **Article: "Introduction to Euro Nymphing"** — 2000+ word original article with full HTML content

### New Pages Built
- **Species list** (`/species`) — grid of 35 species, filterable by family (trout/salmon/char/grayling/saltwater/warmwater/pike)
- **Species detail** (`/species/[slug]`) — hero image, taxonomy table, conservation status, habitat, preferred flies, tackle recommendations, spawning info, fun facts, distribution map coordinates, related destinations/rivers, community photos, photo submission form

### New Components Built
- `GoogleReviews` — displays Google Places reviews with star ratings; graceful empty state when no API key or no reviews. Integrated into lodges, guides, fly shops.
- `CommunityPhotos` — client component that queries Supabase for approved photos by entity type/id, displays in grid with lightbox. Integrated into ALL entity detail pages.
- `PhotoLightbox` — fullscreen photo viewer with EXIF metadata display, prev/next navigation
- `PhotoSubmissionForm` — client component with file upload to Supabase Storage, EXIF metadata fields (camera, lens, aperture, shutter speed, ISO), caption. Requires Supabase auth. Integrated into ALL entity detail pages.

### New API Routes
- `GET /api/google-reviews` — proxy to Google Places API, returns reviews for a given `placeId`
- `POST /api/photos/submit` — handles photo upload metadata, sends admin notification email via Resend
- `GET /api/photos/review` — validates HMAC token, approves/rejects photo, sends confirmation email via Resend

### New Scripts
- `scripts/seed-all.ts` — reads all `src/data/*.ts` files, transforms camelCase keys to snake_case, upserts into Supabase tables. Run with `npm run seed`.
- `scripts/generate-logo.mjs` — generates SVG logo

### Bug Fixes
- **Species type error** — original 8 species entries were missing `relatedDestinationIds`, `relatedRiverIds`, `featured` fields required by the `Species` interface. Rewrote entire file with 35 complete entries.
- **Resend build crash** — `new Resend(process.env.RESEND_API_KEY)` at module top level crashed when env var was missing during build. Fixed with lazy initialization: `getResend()` returns `null` when key is absent, called with optional chaining `resend?.emails.send()`.
- **Homepage adjacent same-color sections** — Lodges section had `bg-cream` same as Species above it. Changed Lodges to `bg-white`, Articles to `bg-cream`.
